# M2-BUG-001: Pagination pages enqueued but not executed

**Status:** OPEN  
**Severity:** High  
**Milestone:** M2 — Raw Scraper  
**Created:** 14 July 2026  
**Module:** `packages/scraping-core/src/orchestrator.ts`

## Observed Behavior

When a category page returns `isNext=true`, the orchestrator enqueues the next page URL into the Crawlee request queue. However, in certain execution contexts the enqueued page is never fetched/executed:

- The discovery crawler's `requestHandler` calls `addRequests(nextPageRequests)` for the next page.
- The enqueued page may not be processed before the crawler run completes (e.g. concurrency limits, early termination, or request queue draining behavior).

The result is that `pagesProcessed` may be N while N+1 pages were logically expected. The category audit remains `completed=false` with `failureKind=undefined`.

## Current Impact

- **Status determination**: Correctly returns `PARTIALLY_FAILED` when `totalDiscovered > 0` (via the `!allCategoriesCompleted` check). Does NOT incorrectly return `SUCCEEDED`.
- **Data completeness**: Only first-page (or a subset of) products are discovered and fetched. Full pagination coverage is not achieved.
- **M3 blocker**: Every enabled category must reach a terminal page (completed=true) before M3 work begins.

## Acceptance Criteria

1. All enabled categories reach a terminal page where `isNext=false` or a recognized terminal state (`PAGINATION_LOOP`, `PAGE_LIMIT_EXCEEDED`).
2. `completed=true` is persisted for every category that reaches a terminal page.
3. Final status is derived correctly: `SUCCEEDED` only when all categories are complete with no terminal failures.
4. Full pagination coverage is verified before M3 normalization/matching work starts.

## M3 Blocker

M3 is blocked until every enabled category reaches terminal pagination. No normalization, classification, or matching work may proceed on partially-discovered product sets.

## Workaround

The deferral policy treats current crawling behavior as first-page/sample only. Structured warnings are emitted for FULL/CATEGORY runs indicating this limitation. The `completed=false` flag on category audits signals incomplete pagination.
